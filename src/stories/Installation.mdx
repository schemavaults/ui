# Installation & Setup

## 1. Install packages

Use your package manager to download the packages from the registry:
```bash
bun install @schemavaults/ui @schemavaults/theme tailwindcss@3.4.17 postcss@8.5.6 autoprefixer@10.4.21
```

## 2. Setup TailwindCSS & generate your configuration using `@schemavaults/theme`

You'll have to configure [TailwindCSS](https://v3.tailwindcss.com/) and [PostCSS](https://postcss.org/), then configure them with `@schemavaults/theme`!

### 2a. Create a TailwindCSS configuration file: `tailwind.config.ts`:
In your app's root directory, create a file named `tailwind.config.ts` that generates a TailwindCSS config:
```typescript
import type { Config } from "tailwindcss";
import { SchemaVaultsTailwindConfigFactory } from "@schemavaults/theme";
const config = new SchemaVaultsTailwindConfigFactory().createConfig({
  content: ["./src/**/*.{tsx,jsx,js,ts,mdx}", "@schemavaults/ui"]
});
export default config;
```

### 2b. Create a PostCSS configuration file at `postcss.config.cjs`:
In your app's root directory, create a file named `postcss.config.cjs` that generates a PostCSS config:
```javascript
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

### 2c. Import `globals.css`
Inside of your app (for example, within your `App.tsx` or root `layout.tsx`) import `globals.css` to trigger PostCSS/TailwindCSS to run:

```jsx
import '@schemavaults/theme/globals.css'

export default function MyApp() {
  return <main><p>Hello World!</p></main>
}
```

### 2d. Additional Steps

Not working? Depending on what type of app environment you're building in you may need additional configuration!

Some app frameworks like Next.js will automatically run PostCSS for you during the build process. Alternatively, other frameworks or custom setups may require you to manually configure PostCSS (which in turn runs TailwindCSS).

- Using webpack to build your app? You'll probably have to use the `postcss-loader` as well. For example, see [the configuration for this Storybook.js site](https://github.com/schemavaults/ui/blob/main/.storybook/main.ts).


## 3. Start building your app!

Components should now work and be properly styled/themed!
